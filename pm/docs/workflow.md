# Workflow (ежедневное использование)

## 1) Захват задач (Inbox)
- Любая мысль/задача сначала попадает в `pm/inbox.md`.
- Можно добавлять “подсказки” прямо в строке:
  - `@project` или `p:project`
  - `#tag`
  - `P0..P3`
  - `due:YYYY-MM-DD`

Пример:
- `поправить webhook P1 @vextaibot #telegram due:2026-02-02`

## 2) Triage (превратить inbox → tasks)
Запускаешь triage — строки превращаются в task-файлы.
- обработанные строки помечаются как `- [x] ... (→ TASK-XXXX)`
- дубликаты помечаются как `(duplicate)`

## 3) Работа (Doing)
- Переводишь 1–3 задачи в `doing`.
- Старайся держать WIP маленьким.

## 4) Завершение (Done)
- Переводишь задачу в `done`.
- По возможности дописываешь краткий результат в `## Лог`.

## 5) Обзор
- `today` / `overdue` для контроля сроков
- Kanban-доска для визуального понимания
