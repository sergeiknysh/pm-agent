# Concept: PM-as-Files

## Что это
PM-as-Files — трекер задач, где **задачи живут в файловой системе** в виде Markdown-файлов.

- **Source of truth:** `pm/projects/**/tasks/*.md`
- Git — история изменений и синхронизация.
- Web Kanban / Telegram / Vexta — всего лишь разные интерфейсы к одному и тому же набору файлов.

## Почему так
- Прозрачные изменения (diff понятен человеку)
- Нет vendor lock-in (файлы — твои)
- Можно редактировать руками и скриптами
- Легко бэкапить/синхронизировать (git)

## Ключевые принципы
- **Одна задача = один файл**
- Метаданные задачи — в **YAML frontmatter** (вверху файла)
- Любые изменения должны:
  1) обновлять `updated`
  2) быть атомарными (write-to-temp + rename)
  3) оставлять лог (в `## Лог`, по возможности)

## Представления
- **Kanban** = группировка по `status`
- **Today/Overdue** = фильтр по `due`
- **Inbox** = сырые входящие строки, которые позже превращаются в задачи
